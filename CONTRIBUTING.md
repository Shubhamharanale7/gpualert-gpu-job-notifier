# Contributing to GPUAlert

See [docs/contributing.md](docs/contributing.md) for the full guide:
dev setup, the four local gates (pytest, ruff, black, `local_test.py`),
style, and the invariants to preserve when changing the package.

For first-time contributors: small fixes (typos, docs, error messages,
new error patterns in `parse_errors.py`) can go straight to a PR. For
larger changes — a new notifier backend, a new CLI command, a config
schema change — open an issue first so we can agree on the shape before
you spend time on it.

Bug reports and feature requests use the templates in
[`.github/ISSUE_TEMPLATE/`](.github/ISSUE_TEMPLATE).
